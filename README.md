# GetSoluce2

This repository contains a small HR management demo built with Laravel. It exposes different dashboards depending on the authenticated user's role (creator, admin or employee).

## Requirements
- PHP 8.1+
- Composer
- SQLite or another supported database

## Setup
1. Clone the repository and install dependencies:
   ```bash
   composer install
   ```
2. Copy `.env.example` to `.env` and adjust database settings.
3. Generate an application key:
   ```bash
   php artisan key:generate
   ```
4. Run migrations and seed the database:
   ```bash
   php artisan migrate --seed
   ```

The included seeders create a demo enterprise, a creator account, an admin account and a regular employee. A sample absence record is also inserted. You can run seeders separately with:
```bash
php artisan db:seed
```

## Running
Start the development server with:
```bash
php artisan serve
```
Then visit `http://localhost:8000`.

### Authentication & Roles

Three roles exist:

- **creator** – manages multiple enterprises and can invite admins.
- **admin** – manages one enterprise and validates requests.
- **employee** – regular user with personal dashboard.

Use the credentials generated by the seeders or register a new account. After logging in you will be redirected to `/dashboard` which then redirects according to the role.

### Running Tests

Feature tests cover role-based access. Execute them with:

```bash
php artisan test
```

Tests require PHP and Composer installed on your system.
